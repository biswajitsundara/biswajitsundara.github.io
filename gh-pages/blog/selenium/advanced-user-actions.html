Advanced User Actions

by Biswajit Sundara

Posted on April 12th, 2020 at 2:00 PM

What is Advanced User Actions

We know that selenium is a browser automation tool that automates user actions like click, enter text etc however the modern applications require advanced user actions such as drag and drop, right click, mouse hover etc. So in order to automate these sophisticated user actions selenium provides the Advanced User Actions API.
  • For example, if you go to flipkart site, Electronics -> Mouse over -> You will see a flying menu is appearing. This is JQuery UI and it can't be handled using the primitive click methods. We need advanced approach to interact.
  • The Actions class from package org.openqa.selenium.interactions is used for the advanced user actions automation.
  • Actions = Class & Action = interface

How it works

//First configure the series of user actions
Actions builder = new Actions(driver);
builder.keyDown(Keys.CONTROL).click(item1)
       .click(item2)
       .click(item3)
       .click(item5)
       .keyUp(Keys.CONTROL);

//Get the action/built the object to execute
Action selectMultiple = builder.build();
		
//Execute the action object
selectMultiple.perform();
The builder design pattern is used here so instead of creating the action object & then calling the perform method we can straight forward way use .build() & .perform().



Mouse Events

The mouse interface deals with advanced user actions those can be performed using mouse. Let's look into all the methods one by one.

1. Click

The Click method is overloaded. click() without argument clicks on the last known mouse coordinates & the click(WebElement element) clicks on the middle of the given element. We need to use the web driver click method for the normal usage and the correct usage of this method is while clicking any menu item from the hover menu. First we will hover on the flyout menu & then click on the sub menu item. click(WebElement element) = Actions.moveToElement(onElement).click(). 
public void mouseClick(WebElement element)
{
    Actions builder = new Actions(driver);
    builder.click(element).build().perform();
}

2. Double Click

The doubleClick() without argument double clicks on the last known mouse coordinates & the doubleClick(WebElement element) double clicks on the middle of the given element. doubleClick(WebElement element) = Actions.moveToElement(element).doubleClick(). 
public void mouseDoubleClick(WebElement element)
{
    Actions builder = new Actions(driver);		
    builder.doubleClick(element).build().perform();
}

3. Right Click

To perform right click, the method name is contextClick. contextClick() without argument performs the right click operation on the last known mouse coordinates & the contextClick(WebElement element) right clicks on the middle of the given element. Ever wondered why selenium calls it context click instead of right click? that's because when we perform right click on UI the menu that appears is called context menu. 
public static void mouseContextClick(WebElement element)
{
    Actions builder = new Actions(driver);
    builder.contextClick(element).build().perform();
}

4. Click and Hold

There are two versions of this method. clickAndHold(WebElement element) clicks on the middle of the given element without releasing the button and clickAndHold() clicks and holds on the last known coordinate. 
public void mouseClickAndHold(WebElement element)
{
    Actions builder= new Actions(driver);
    builder.clickAndHold(element).build().perform();
}

5. Mouse Release

There are two versions of this method. release(WebElement element) releases the pressed left mouse button on the middle of the given element. release() releases the pressed left mouse button at the current mouse location. First we need to call the clickAndHold() method then the release() method else it will end up in undefined behaviour. 
public void mouseButtonRelease(WebElement element)
{
    Actions builder= new Actions(driver);
    builder.release(element).build().perform();
}

6. Mouse Hover

Moves the mouse to the middle of the element. The element is scrolled into view and its location is calculated using getBoundingClientRect. 
public void mouseHover(WebElement element)
{
    Actions builder= new Actions(driver);
    builder.moveToElement(element).build().perform();
}

7. Drag & Drop

It clicks-and-holds at the location of the source element then moves to the target location and releases the mouse there. 
public void mouseDragAndDrop(WebElement source, WebElement target)
{
    Actions builder= new Actions(driver);
    builder.dragAndDrop(source, target).build().perform();
}

8. Drag & Drop By Coordinates

It clicks-and-holds at the location of the source element then moves to specified mouse location and releases the mouse there. 
public void dragAndDropByCoord(WebElement element, int xaxis, int yaxis)
{
     Actions builder= new Actions(driver);
     builder.dragAndDropBy(element, xaxis, yaxis).build().perform();
}




Keyboard Events

The keyboard events in Actions class helps to simulate the user actions of pressing a single or multiple keys on the keyboard. We do have the send keys method in webdriver using which we can pass the keyboard keys however the advanced actions like pressing a key and then performing some action is not possible there. So we use the Keyboard events from Actions class.

1. Send Keys

The sendKeys(WebElement element, CharSequence... keys) first clicks on the element and then enters the keys. There is another version sendKeys() which sends the key to the active element. There can be a question, what is the difference between actions class sendKeys() & the selenium web driver sendKeys. There are two differences
  • The modifier keys included in this call are not released. For example if we press Ctrl, Shift keys then thoese don't get released after the sendKeys we will have to release them by calling keyUp()
  • There is no attempt to re-focus the element - so sendKeys(Keys.TAB) for switching elements should work. 
public void sendKeyAction(WebElement element, String text)
{
    Actions builder= new Actions(driver);
    builder.sendKeys(element, text).build().perform();
}

2. KeyDown

The keyDown(WebElement element,CharSequence key) first clicks on the element and then press the modifier key. The modifier key means control/shift/alt. Please note two things
  • It doesn't release the key. For example, if we have used the control key then visualize it as the key remains pressed on the keyboard. To release the key we need to use keyUP(CharSequence key) method.
  • Also the valid keys are control/shift/alt. Any other key used here would end up with IllegalArgumentException.
The other version of the method keyDown(CharSequence key) can be used if we don't have to focus on any element. 
public void sendKeyAction(WebElement element, String text)
{
    Actions builder= new Actions(driver);
    builder.sendKeys(element, text).build().perform();
}

3. KeyUp

The keyUp(CharSequence key) releases the modifier key. The modifier key means control/shift/alt. Please note two things
  • The keyDown(CharSequence key) keeps the key pressed and the keyUp(CharSequence key) releases the modifier key. To release the key we need to use keyUP(CharSequence key) method.
  • The valid keys those can be used here are control/shift/alt. Any other key used here would end up with IllegalArgumentException.
The other version of the method keyUp(WebElement, CharSequence key) can be used if we have to focus on any element before releasing the key. 
public void keyUpAction(WebElement element1)
{
    Actions builder= new Actions(driver);
    builder.keyDown(element1, Keys.CONTROL)
           .keyUp(Keys.CONTROL)
           .build().perform();
}




FlipKart

If you open flipkart, you will see the apple sub menu is not visible and we can't click on it without moving mouse over the electronics menu. So let's open the flip kart site, hover on the electronics fly out menu & then select the Apple sub menu using selenium action class. 

public class FlipKart 
{
	
     private WebDriver driver;

	
     @BeforeClass
     public void setup()
     {
	//Instantiate WebDriver
	System.setProperty("webdriver.chrome.driver", "C:\\drivers\\chromedriver.exe");
	driver= new ChromeDriver();
	driver.manage().window().maximize();
	driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS);
		
	//Launch the flipkart site & close the login popup
	driver.get("https://www.flipkart.com/");
	driver.findElement(By.xpath("//button[@class='_2AkmmA _29YdH8']")).click();
	System.out.println("The flipkart site opened successfully");
     }


     @Test
     public void testFlipkart()
     {

        //Instantiate the action class
	Actions builder = new Actions(driver);

        //Locate the menu & sub menu item
	WebElement electronics = driver.findElement
                                 (By.xpath("//span[text()='Electronics']"));
	WebElement appleMenu = driver.findElement(By.xpath("//a[text()='Apple']"));


        //First hover on the fly out menu & then hover on sub menu & click
	builder.moveToElement(electronics)
	       .click()
	       .moveToElement(appleMenu)
	       .click()
	       .perform();	
      }


      @AfterClass
      public void tearDown()
      {
	 //driver.quit();
      }
	
}

Select Multiple

To select multiple items, we need to first click on an item then press the control key & select other items. Below we have used the JQuery site for the demo. 

public class MultipleSelect 
{

    private WebDriver driver;

    @BeforeClass
    public void setup()
    {

	//Instantiate WebDriver
	System.setProperty("webdriver.chrome.driver", "C:\\drivers\\chromedriver.exe");
	driver= new ChromeDriver();
	driver.manage().window().maximize();
	driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS);

        //Launch the flipkart site & close the login popup
	driver.get("https://jqueryui.com/selectable/");
	WebElement frame = driver.findElement(By.className("demo-frame"));
	driver.switchTo().frame(frame);
	System.out.println("The J Query site opened successfully");

    }


    @Test
    public void testMultipleSelect()
    {

	//Locate the web elements
	WebElement item1 = driver.findElement(By.xpath("//li[text()='Item 1']"));
	WebElement item2 = driver.findElement(By.xpath("//li[text()='Item 2']"));
	WebElement item3 = driver.findElement(By.xpath("//li[text()='Item 3']"));
	WebElement item5 = driver.findElement(By.xpath("//li[text()='Item 5']"));


	//press the control key & click on items.
	Actions builder = new Actions(driver);
		builder.keyDown(Keys.CONTROL).click(item1)
		.click(item2)
		.click(item3)
		.click(item5)
		.keyUp(Keys.CONTROL)
		.build()
		.perform();

    }


     @AfterClass
     public void tearDown()
     {
	//driver.quit();
     }

}




Native Events vs Synthetic Events

In WebDriver advanced user interactions are provided by either simulating the Javascript events directly (i.e. synthetic events) or by letting the browser generate the Javascript events (i.e. native events). Native events simulate the user interactions better whereas synthetic events are platform independent. Native events should be used whenever it is possible.

ChromeDriver provides two modes of supporting native events called WebKit events and raw events. In the WebKit events the ChromeDriver calls the WebKit functions which trigger Javascript events, in the raw events mode operating systems events are used.

In the FirefoxDriver, native events can be turned on and off in the FirefoxProfile. 
FirefoxProfile profile = new FirefoxProfile();
profile.setEnableNativeEvents(true);
FirefoxDriver driver = new FirefoxDriver(profile);
These are some examples where native events behave different to synthetic events:
  • With synthetic events it is possible to click on elements which are hidden behind other elements. With native events the browser sends the click event to the top most element at the given location, as it would happen when the user clicks on the specific location.
  • When a user presses the 'tab' key the focus jumps from the current element to the next element. This is done by the browser. With synthetic events the browser does not know that the 'tab' key is pressed and therefore won't change the focus. With native events the browser will behave as expected.

Summary

  • The Action class is less stable. So always prefer to use selenium web driver methods at the first place, if we are not able to perform the required action using selenium wd, then only go for the advanced user interactions API.
  • Firefox supports both Native events & Synthetic events so the assumption is that for these advanced user actions, firefox can be the preferred browser.
  • Chrome & Internet explorer support native events but don't support synthetic events.
  • Be cautious while using action class in headless mode.
