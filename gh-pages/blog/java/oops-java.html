OOPS Basics

Object Oriented Programming (Paradigms) is a way of writing programs. The programming aims to resemble to the real world. Here everything is based on classes and objects.


Why do we need OOPS

  • OOPS provides a clear modular structure to the programming.
  • It's easy to build and modify complex codes.
  • Code reusability
  • As it resembles the real world, easy to solve the real world problems through this.
  • It is secured. Unlike the procedural language OOPS ensures security.

Languages support OOPS

  • Simula (The first OOPS language)
  • Java
  • Ruby
  • Python
  • C++ etc




Class & Objects

This can be defined in two ways. We can say class is a blue print or proto type from which objects are created. For example: Employee class may contain data memebrs like name, age, salary, dept etc & the methods like swipeIn(), swipeOut(), getSalary() etc. Now this is a blue print, whenever we create any employee object, the object will have a copy of all this. The employee will have a set of attributes like name, age etc and then can access the methods.

Or we can say class is a collection of similar objects. For example - Employee Mark, Employee Jack & Employee Jill are objects of one class Employee. Class also represents the set of properties or methods that are common to all objects. For example - all employees will have properties/attributes such as name, employee id, salary etc. methods such as markattendance(), getSalary(), applyLeave() etc.

Components of Class

  • Modifiers : A class can be public or has default access
  • Class name: The name should begin with a initial letter (capitalized by convention).
  • Superclass(if any): The name of the parent class (if any) preceded by the extends keyword. In Java a subclass can have only one parent class.
  • Interfaces(if any): The name of interface(interfaces separated by ,) preceded by the implements keyword. It can implement multiple interfaces
  • Body: The class body surrounded by braces, { }.
Constructors are used for initializing new objects. Fields are variables that provides the state of the class and its objects, and methods are used to implement the behavior of the class and its objects.

Object

Objects are nothing but instance of the class. A single class can create any number of unique objects. An object consists of
  • State : It is represented by attributes of an object. It also reflects the properties of an object.
  • Behavior : It is represented by methods of an object. It also reflects the response of an object with other objects.
  • Identity : It gives a unique name to an object and enables one object to interact with other objects.


In the above example - Identity is the roll number of the student. State/Attributes are - Name, age, color, caste etc. Behaviors - Entering into class, leaving the class, writing assignments, asking questions etc.

Object Declaration

When an object of a class is created, the class is said to be instantiated. All the instances share the attributes and the behavior of the class. But the values of those attributes, i.e. the state are unique for each object. A single class may have any number of instances. In general,we can't create objects of an abstract class or an interface. 

Student Priyanka;
If we declare reference variable(Priyanka) like this, its value will be undetermined(null) until an object is actually created and assigned to it. Simply declaring a reference variable does not create an object.

Object Initialization

The new operator instantiates a class by allocating memory for a new object and returning a reference to that memory. The new operator also invokes the class constructor. There are three ways we can initialize object - by reference variable, by method, by constructor The below program Student.java defines a class, creates objects and gives the following output.

My rollnumber, age and color are 10257,29,white
This class contains a single constructor. We can recognize a constructor because its declaration uses the same name as the class and it has no return type. The Java compiler differentiates the constructors based on the number and the type of the arguments. The constructor in the Student class takes four arguments. The following statement provides "Priyanka",10257,29,"white" as values for those arguments:

Different ways to Create Object

There are many ways to create an object in java.
  • By new keyword
  • By newInstance() method
  • By clone() method
  • By deserialization
  • Factory Method
1. New keyword
It is the most common and general way to create object in java. 

public class NewKeywordExample  
{

    { 
        // Here we are creating Object of  
        // NewKeywordExample using new keyword

    } 
}
2. Using New Instance
If we know the name of the class & if it has a public default constructor we can create an object –Class.forName. We can use it to create the Object of a Class. Class.forName actually loads the Class in Java but doesn’t create any Object. To Create an Object of the Class you have to use the new Instance Method of the Class. 

public class NewInstanceExample 
{ 
    String name = "HelloWorld"; 
    public static void main(String[] args) 
    { 
        try
        { 
            Class cls = Class.forName("NewInstanceExample"); 
            NewInstanceExample obj = (NewInstanceExample) cls.newInstance(); 
            System.out.println(obj.name); 
        } 
        catch (ClassNotFoundException e) 
        { 
            e.printStackTrace(); 
        } 
        catch (InstantiationException e) 
        { 
            e.printStackTrace(); 
        } 
        catch (IllegalAccessException e) 
        { 
            e.printStackTrace(); 
        } 
    } 
}
3. Using Clone Method
Whenever clone() is called on any object, the JVM actually creates a new object and copies all content of the previous object into it. Creating an object using the clone method does not invoke any constructor. To use clone() method on an object we need to implement Cloneable and define the clone() method in it. 
public class CloneExample implements Cloneable 
{ 
    @Override
    protected Object clone() throws CloneNotSupportedException 
    { 
        return super.clone(); 
    } 
    String name = "GeeksForGeeks"; 
  
    public static void main(String[] args) 
    { 
        CloneExample obj1 = new CloneExample(); 
        try
        { 
            CloneExample obj2 = (CloneExample) obj1.clone(); 
            System.out.println(obj2.name); 
        } 
        catch (CloneNotSupportedException e) 
        { 
            e.printStackTrace(); 
        } 
    } 
}

//Note : Here we are creating the clone of an existing Object and not any new Object.
//Class need to implement Cloneable Interface otherwise it will throw CloneNotSupportedException.
4. Using Deserialization
Whenever we serialize and then deserialize an object, JVM creates a separate object. In deserialization, JVM doesn’t use any constructor to create the object. To deserialize an object we need to implement the Serializable interface in the class. 

//Serializing an Object :
class DeserializationExample implements Serializable 
{ 
    private String name; 
    DeserializationExample(String name) 
    { 
        this.name = name; 
    } 
  
    public static void main(String[] args) 
    { 
        try
        { 
            DeserializationExample d = 
                    new DeserializationExample("GeeksForGeeks"); 
            FileOutputStream f = new FileOutputStream("file.txt"); 
            ObjectOutputStream oos = new ObjectOutputStream(f); 
            oos.writeObject(d); 
            oos.close(); 
            f.close(); 
        } 
        catch (Exception e) 
        { 
            e.printStackTrace(); 
        } 
    } 
}
Object of DeserializationExample class is serialized using writeObject() method and written to file.txt file. 

//Deserialization of Object :

        {

5. Using newInstance() method of Constructor class
This is similar to the newInstance() method of a class. There is one newInstance() method in the java.lang.reflect.Constructor class which we can use to create objects. It can also call parameterized constructor, and private constructor by using this newInstance() method. Both newInstance() methods are known as reflective ways to create objects. In fact newInstance() method of Class internally uses newInstance() method of Constructor class. 

public class ReflectionExample 
{ 
    private String name; 
    ReflectionExample() 
    { 
    } 
    public void setName(String name) 
    { 
        this.name = name; 
    } 
    public static void main(String[] args) 
    { 
        try
        { 
            Constructor constructor 
                = ReflectionExample.class.getDeclaredConstructor(); 
            ReflectionExample r = constructor.newInstance(); 
            r.setName("GeeksForGeeks"); 
            System.out.println(r.name); 
        } 
        catch (Exception e) 
        { 
            e.printStackTrace(); 
        } 
    } 
}




Different Type Of Classes


Nested Class

Java allows us to define a class within another class. Such a class is called a nested class. 
    
class OuterClass 
{
    ...
    class NestedClass 
    {
        ...
    }
}

Why Use Nested Classes?

  • It is a way of logically grouping classes that are only used in one place: If a class is useful to only one other class, then it is logical to embed it in that class and keep the two together. Nesting such "helper classes" makes their package more streamlined.
  • It increases encapsulation: Consider two top-level classes, A and B, where B needs access to members of A that would otherwise be declared private. By hiding class B within class A, A's members can be declared private and B can access them. In addition, B itself can be hidden from the outside world.
  • It can lead to more readable and maintainable code: Nesting small classes within top-level classes places the code closer to where it is used.
  • Java doesn't allow a class to be declared as private. However in nested class architecture we can declare the inner class as private and it will not be accessed out side of the outer class.
  • The inner class can access the private or public variables/methods of the outer class.
  • Nested classes are of three types - Inner class, Method Local Inner Class, Static nested classes, Anonymous Inner Class

1. Inner Class

In nested class architecture the class that is inside the outer class is called inner class. The class can be private and it can't be accessed from an object outside of the class. As a side note, we can't have static method in a nested inner class because an inner class is implicitly associated with an object of its outer class so it cannot define any static method for itself.

}
public class NestedClassDemo

}

2. Method Local Inner Class

Inner class can be declared within a method of an outer class. Method Local inner classes can’t use local variable of outer method until that local variable is not declared as final. The main reason we need to declare a local variable as a final is that local variable lives on stack till method is on the stack but there might be a case the object of inner class still lives on the heap. Method local inner class can’t be marked as private, protected, static and transient but can be marked as abstract and final, but not both at the same time. 

class OutClass
{
	public void OutMethod()
	{
		class InnerClass
		{

			{
				System.out.println("Method inner class..");
			}
		}

		incls.innerMethod();
	}
}
public class Method_Local_Inner_Class {

5. Pojo Class

Pojo stands for Plain Old Java Object. This class is similar to any other class in Java however it has some set of rules. The reason of having this class is to simplify the web development process. Let's say we have a pojo class for employee. The class has set of attributes and methods. When employee details are fetched from DB. It will be done through the pojo class object and then using the same object the web page can access the values. A question may arise then what is Java Bean. Java Bean has also the similar features, so we can say all Java Bean classes are POJO but not all POJOs are Java Bean.
  • A Pojo class shouldn't extend any class
  • A Pojo class shouldn't implement any interface
  • The class must be public i.e accessible to all.
  • Properties or variables defined in the class must be private i.e. can't be accessed directly.
  • The class must have default constructor i.e no argument constructor.
  • Public Getter and Setter must be present in the classes in order to access the variables/properties. 
class Employee

	private String id;

		this.name = name;

		this.id = id;
	}

	public double getSalary() {
		return salary;
	}

	public void setSalary(double salary) {
		this.salary = salary;
	}
}

Inheritance

Inheritance simply means Relationship between classes.
  • Inheritance is the mechanism where one class acquires the properties of another class.
  • In other way, we can say this is an approach where new class is derived from an existing class.
  • Whenever Inheritance comes, just visualize the child-parent relationship. The child inherits the properties of the parent.
  • Similarly child classes may inherit or acquire the properties and methods of the parent classes.
  • The child class often referred as sub class, derived class etc, the parent class is often referred as superclass etc.
  • The keyword extends is used to indicate the child class extends the parent class.
  • In Java if we don't declare any super class/parent class then it extends the Object Class
  • That's why it's said, Object Class is the root of all inheritance hierarchies.
  • And it's the only class in Java that doesn't extend another class. 
public class Subclass extends Superclass
{
	//methods and fields
}

Why do we need Inheritance?

We need inheritance for code reusability. Some people say inheritance is required for polymorphism(overloading or overriding) however that's not right. Code reusability is the correct answer.

Let's discuss a scenario here. We have car classes such as - BMW, Maruti, i20 etc. All cars will have some basic features like start engine, fuel, key, accelerator etc. So if we design it as it is then we will have to repeat the same properties/methods in all the car classes. Instead of that we can create a car class and have all the common properties defined there. Then each car class BMW, Maruti, i20 etc can extend the car class and use the properties. This way we achieve code reusability.


Types Of Inheritance?

  • There can be three types of inheritance in java- Single, Multilevel and Hierarchical. Using classes.
  • There are two more types - Multiple and Hybrid inheritance which are supported through interface only.

Single Inheritance

Single inheritance enables a derived class to inherit properties and behavior from a single parent class. It allows a derived class to inherit the properties and behavior of a base class, thus enabling code reusability as well as adding new features to the existing code. This makes the code much more elegant and less repetitive. Single inheritance is safer than multiple inheritance if it is approached in the right way. It also enables a derived class to call the parent class implementation for a specific method if this method is overridden in the derived class or the parent class constructor.

Example: Here the architecture is DerivedClass --> BaseClass. Let's say iPhone inherits the properties of ApplePhones. 

//base class 
class ApplePhones
{ 
	// the apple class has two fields 
	public String OS; 
	public int version; 

	// the apple class has one constructor 
	public ApplePhones(int version) 
	{ 
		OS = "IOS"; 
		this.version = version; 
	} 

	public String getOSDetails() 
	{ 
		return OS; 
	} 

	public void updateVersion(int increment) 
	{ 
		version = version+increment; 
	} 

} 

//derived class 
class IPhone extends ApplePhones 
{ 

	public String product_name;
	public int Price;

	// the IPhone subclass has one constructor 
	public IPhone(int version, String productname, int price) 
	{ 
		// invoking base-class(AplePhones) constructor 
		super(version); 
		product_name= productname; 
		Price= price;
	}  

	// overriding getOS
	@Override
	public String getOSDetails() 
	{ 
		return "iOS001"; 
	} 

} 

public class SingleInheritanceDemo {

	public static void main(String[] args) 
	{
		IPhone iphone = new IPhone(1,"iPhone MX", 50000); 
		System.out.println("ProductName=" +iphone.product_name);
		System.out.println("OS=" +iphone.OS);
		System.out.println("version=" +iphone.version);
		System.out.println("Price=" +iphone.Price);

		IPhone iphone2 = new IPhone(2,"iPhone XX", 90000); 
		System.out.println("ProductName=" +iphone2.product_name);
		System.out.println("OS=" +iphone2.getOSDetails());
		iphone2.updateVersion(2);
		System.out.println("version=" +iphone2.version);
		System.out.println("Price=" +iphone2.Price);
	}
}

Multiple Inheritance

When a class inherits properties of more than one parent class, that is called Multiple Inheritance. A common question often asked is, does Java support multiple inheritance? The answer is No. Why not? The problem occurs when there exist methods with same signature in both the super classes and subclass. On calling the method, the compiler cannot determine which class method to be called and even on calling which class method gets the priority. We can implement multiple inheritance using interfaces.

  • Object is the superclass of all classes in java.
  • Default superclass: Except Object class, which has no superclass, every class has one and only one direct superclass (single inheritance). In the absence of any other explicit superclass, every class is implicitly a subclass of Object class.
  • Superclass can only be one: A superclass can have any number of subclasses. But a subclass can have only one superclass. This is because Java does not support multiple inheritance with classes. Although with interfaces, multiple inheritance is supported by java.
  • Inheriting Constructors: A subclass inherits all the members (fields, methods, and nested classes) from its superclass. Constructors are not members, so they are not inherited by subclasses, but the constructor of the superclass can be invoked from the subclass.
  • Private member inheritance: A subclass does not inherit the private members of its parent class. However, if the superclass has public or protected methods(like getters and setters) for accessing its private fields, these can also be used by the subclass.

What all can be done in a Subclass?

In sub-classes we can inherit members as is, replace them, hide them, or supplement them with new members:
  • The inherited fields can be used directly, just like any other fields.
  • We can declare new fields in the subclass that are not in the superclass.
  • The inherited methods can be used directly as they are.
  • We can write a new instance method in the subclass that has the same signature as the one in the superclass, thus overriding it (as in example above, toString() method is overridden).
  • We can write a new static method in the subclass that has the same signature as the one in the superclass, thus hiding it.
  • We can declare new methods in the subclass that are not in the superclass.
  • We can write a subclass constructor that invokes the constructor of the superclass, either implicitly or by using the keyword super.

Cohesion

The dictionary meaning of cohesion is keeping it united as a whole. Applying the same principle in object oriented software design cohesion is defined as the purpose of a class. A well focused class designed for a particular purpose is considered high cohesion and a less focused class is considered low cohesion. The best practice is to have high cohesion always.

Below is an example of low cohesion. The Employee is assigned with email tasks however there's another task to print letters also and that's for another purpose. 
public class Employee
{
  checkEmail()    
  sendEmail()     
  emailValidate() 
  PrintLetter()   
}

Below is an example of high cohesion. The HR Employee is assigned with tasks related to calculating salary, then set salary to employees and transfer salary to employee. Here all tasks are focused to a single purpose. 
public class HR Employee
{
  calculateSalary()
  setSalary()    
  transferSalary()       
}

Key Points

  • Cohesion is the relationship within a module.
  • Cohesion is the degree to which a class/ module focuses on the single thing
  • While designing we should strive for high cohesion i.e class with focused to a single task
Cohesion may sound similar to Single Responsibility Pattern (SRP) that's because when a class is designed to have a single responsibility then it has high cohesion.

Association

Association defines relationship between two separate classes through their objects.
  • Association defines a relationship between two class objects based on common attributes.
  • The association can be one-to-one. one-to-many, many-to-many
  • Composition and Aggregation are the two types of association.
  • Let's take an example, bank and employee.
  • There are multiple banks an employee can have. one bank can have multiple employees also
  • Both have their own life cycles (they can be created or deleted independently)
  • Let's say if one employee is deleted, it doesn't impact the bank.
  • Similarly if a bank is deleted it doesn't impact the employee.

Aggregation

Aggregation is a type of association between two classes.
  • Aggregation defines HAS A relationship between classes.
  • For example, Employee class HAS A address class.
  • This relationship is unidirectional means employee class has address but vice versa is not possible.
  • One class can exist without another class means if we nullify the object of employee class, the address object still exists
  • Aggregation can be used instead of inheritance however we will have to see the need for so.

Composition

Composition is a restricted form of Aggregation in which two entities are highly dependent on each other. When an object contains the other object, if the contained object cannot exist without the existence of container object, then it is called composition.
  • It is Part-Of relationship.
  • An employee contains address. Means Address is part of Employee class.
  • In composition, both the entities are dependent on each other
  • When there is a composition between two entities, the composed object cannot exist without the other entity.
  • A good design pattern is to use composition over inheritance.
  • A class contains students. A student cannot exist without a class. There exists composition between class and students.
  • This restriction is not in aggregation. Though one object can contain the other object, there is no condition that the composed object must exist.
  • A Library contains students and books. Relationship between library and student is aggregation because a student can exist without a library but A book can not exist without a library so the relation between books and library is called composition.
