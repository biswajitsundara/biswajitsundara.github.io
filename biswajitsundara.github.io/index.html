Test Automation Basics

What is Test Automation?

The true meaning of growing is to get transformed into something better and beautiful. Each period of history tells us how things have evolved over time. Today the need of the hour is automation. We need to save human efforts where ever we can so that gets utilized in something else. The core idea of automation is to save efforts and reduce cost of the project.

When test cases shouldn't be automated?

There is a misconception that manual testing will be replaced by automation testing which is not true. Read below to find out when we should go for manual testing instead of automation.
  • Static tables: Data is static i.e. Number of rows and columns are fixed.
  • Dynamic tables: Data is dynamic i.e. Number of rows and columns are NOT fixed.

What should be automated?

  1. Regression Testing Suite should always be automated as this is something we execute repeatedly. There's a misconception that regression testing is 100% automated always which is not true. It depends on the application however mostly it's fully automated.
  2. Smoke Testing Suite should be automated. When the build completes it can execute automatically and give a go/ no go decision for the deeper testing.
  3. Any test that is repitive and relatively unchanging should always be automated.
  4. Data driven testing, where the same functions need to be validated with lot of different data input/output. For example, create 100 employees with different data.
  5. Load/performance testing needs to be automated as we can't check that manually.
  6. Automation Tests give faster test coverage, automated reports, accurate results.
  7. We can have scripts run overnight, review results in the morning and advise development of issues faster
  8. Accuracy of data/calculation is involved. For example, we need to calculate the employee's gross salary considering other data validations.
  9. Compatibility Testing. Same testcases can be executed in different browsers, platforms, environments etc.

When to decide Manual or Automation

While reviewing the requirements, we start discussing what can be automated. While testing if the testing covers any critical functionality that needs to be executed repeatedly each release, then it is automated and added to the regression suite. It's a continuous process and the project team plays an important role.

New Function –> test case –> manual test –> works / passes –> release –> create automated script for regression


How to calculate ROI?
Cost of the tool (licensing), cost of the ongoing maintenace(if any), hours*dollar to be invested to develop the scripts Cost to train team mates.
